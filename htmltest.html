Data and Weights

Our data is stored in data.csv, which contains details about individualsâ€”such as name, undergrad institution, grad school, employer, location, hobbies, and drinks. A second file, weights.csv, defines the relative importance (weights) for each field.

Building the Undirected Knowledge Graph

We construct an undirected knowledge graph by representing each person and each trait (e.g., education, employer, etc.) as nodes. Edges connect people to their traits using weights derived from weights.csv.

Learning Node Embeddings & Repositioning

Next, we learn vector embeddings for each node using a simple machine learning model. These embeddings capture similarities between nodes. We then reposition the person nodes using Multi-Dimensional Scaling (MDS) so that similar people appear closer together. When you hover over a node in the embedded graph below, you'll see a list of other people ordered by closeness along with their similarity scores.

Summary

The above visualizations illustrate our process: We start with data.csv and weights.csv to build an undirected knowledge graph. We then learn node embeddings that capture relationships and reposition the people nodes based on similarity. Hover details provide additional insight into how similar each person is to the others.
